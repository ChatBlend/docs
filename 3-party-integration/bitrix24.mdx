Within the Send Lead to CRM section, each question or variable captured by the bot can be paired with a specific Bitrix24 parameter for seamless synchronization.
 
📸 Figure 6 Birix24 Lead mapping

Actions shown in the configuration:
1.	Question mapping:
The ChatBlend Bot’s question “Do you want to know……?” is mapped to the Bitrix24 field Note, so any user response to that question will be saved in the CRM as a note or, any other from the list.
 




2.	Dynamic atributes mapping:
o	The Language attribute is mapped Source Description, allowing you to track the user’s contact channel.
 
o	The Business Need attribute is mapped to Comment, so context or intent gathered during the conversation is logged directly in the deal or lead comments.

•	Bitrix24 Attribute Mapping: Link Bitrix24 attributes (Deal, Contact, Lead) → ChatBlend variables.
The module enables ChatBlend to synchronize CRM data dynamically, ensuring that key Bitrix24 attributes (such as deal status, tags, or assigned users) stay aligned with your chatbot logic.
This mapping makes it possible for ChatBlend to read and update Bitrix24 records automatically during the conversation flow — maintaining a live connection between both systems.
 
📸 Figure 7 Mapping Bitrix24 CRM attributes to ChatBlend parameters for data synchronization
How it works:
Each row defines a pairing between a ChatBlend attribute and a Bitrix24 parameter.
When ChatBlend sends or receives data from Bitrix24, it uses these mappings to translate field names and keep both sides consistent.
Actions shown in this configuration example:
•	Tags Mapping:
The ChatBlend tag “Rocket” is linked to a Bitrix24 tag field.
Tags help classify leads or deals (e.g., campaign, product line, or interest level) and are automatically created in Bitrix24 if they don’t already exist.
•	Status and Pipeline Mapping:
ChatBlend updates the deal or lead stage in Bitrix24 based on the conversation flow or chatbot logic.
When a customer reaches a key point during the interaction — for example, New Opportunity, Needs Assessment, or Free Trial Setup — ChatBlend automatically updates the corresponding Deal Stage (STAGE_ID) in Bitrix24 through a one-way synchronization.
This configuration keeps the CRM pipeline aligned with the customer’s progress in WhatsApp conversations, ensuring that Bitrix24 always reflects the most recent stage reached through ChatBlend.
Typical examples include moving a deal to Needs Assessment when a prospect confirms interest, or updating to Won when a renewal is completed through chat.
Note: The current integration operates in a single direction (ChatBlend → Bitrix24).

•	ChatBlend and Bitrix24 share aligned stage definitions to ensure both systems reflect the same customer progression.
When a conversation in ChatBlend reaches a milestone (for example, New Opportunity, Needs Assessment, or Free Trial Setup), the integration automatically updates the corresponding Deal Stage (STAGE_ID) or Status in Bitrix24.
Likewise, when a stage is modified in Bitrix24 — for instance, a deal is moved from Needs Assessment to Won — ChatBlend’s Inbox Chat Status instantly mirrors that change.
This two-way synchronization maintains consistency between chat engagement and CRM deal tracking, allowing ChatBlend to deliver real-time visibility and analytics without duplicating CRM functionality.
Bitrix24 remains the system of record for all pipeline operations, while ChatBlend acts as an intelligent conversational layer that mirrors deal progress and supports automated follow-ups and analytics.
Tip: Use identical stage names and hierarchy in both ChatBlend and Bitrix24 (for example, New Opportunity, Needs Assessment, Free Trial Setup, Won, Lost) to ensure accurate synchronization and clean reporting across both platforms.
•	Assigned Users:
The Assigned Users parameter links the lead or deal to a specific Bitrix24 user (in this example, RIPUZ SAS).
This routing ensures that new conversations or deals are automatically assigned to the correct sales or support representative.
•	Saving and Synchronization:
o	Click Update to save your mappings.
o	Click Sync to test live synchronization.
ChatBlend will verify that both systems recognize the linked parameters and confirm successful mapping.





•	Automated Deal Creation: Auto-create Deals directly from chat interactions.
The module allows ChatBlend to generate or update deals in Bitrix24 automatically, based on the answers captured during a chatbot conversation.
Each question defined in ChatBlend can be mapped to a specific Bitrix24 field (identified by its unique UF_CRM_ parameter), ensuring that deal data is continuously enriched as the chat progresses.
 
📸 Figure 8 Automatic deal creation and field mapping from ChatBlend conversation questions to Bitrix24 Deal fields.
How it works:
As the chatbot asks predefined questions (for example, “Could you please share your company name?” or “Let me know what service you need”), the customer’s answers are automatically assigned to their respective Bitrix24 deal fields.
 Each question is linked to a Bitrix24 parameter such as:
•	UF_CRM_1759610251 → Company Name
•	UF_CRM_681784764369 → Service Interest

When the conversation reaches a trigger point (for example, a user confirms interest or shares key details), ChatBlend either creates a new Deal or updates an existing Deal in Bitrix24 using these field values.

The Update button saves all mappings, activating the automation. From that moment, every new conversation that reaches those questions will automatically populate or update the CRM.
Benefits:
•	Eliminates manual deal creation by sales teams.
•	Keeps Bitrix24 always up-to-date with live data captured in chat.
•	Maintains full traceability — each field in Bitrix24 corresponds to a real user input from WhatsApp.

3️⃣ Automation in Bitrix24
You can create workflows or automation rules for any Bitrix24 entity (Leads, Deals, Contacts).
 
🔹 Stage 1 — Technical Visit Request
•	A new Deal / SPA is created in Bitrix24 from a customer request (form, WhatsApp chat, or manual entry).
•	The request includes essential data: client name, contact channel, service type, and initial notes.
•	No automation runs yet — this stage represents registration and triage.

🔹 Stage 2 — Scheduled (Workflow Trigger Point)
•	When the Deal is moved to Scheduled, Bitrix24 triggers the ChatBlend Action → Send WhatsApp Message (Template: FieldOps_Visit).
•	The template includes contextual details and interactive buttons:
🕘 I’ve Arrived · 📍 Send Location · 📝 Report Activity · 📸 Upload Photos · ✅ Job Completed
•	The message is sent automatically to the assigned field agent.
•	The agent’s responses in WhatsApp start updating the Deal through ChatBlend mappings.




🔹 Stage 3 — On-Site
•	When the agent clicks, I’ve Arrived, ChatBlend updates:
•	UF_CRM_FIELD_STATUS = Arrived
•	UF_CRM_CHECKIN_TS = timestamp
•	Bitrix24 automation moves the Deal to On-Site.
•	Optional: a notification or comment is added for the supervisor —
•	“Agent arrived at {{CHECKIN_TS}}.”
🔹 Stage 4 — In-Progress
•	The agent sends location, activity summary, and photos.
•	ChatBlend updates the following fields in Bitrix24:
•	UF_CRM_LAT, UF_CRM_LNG, UF_CRM_ACTIVITY_SUMMARY, UF_CRM_MEDIA_FILES.
•	Bitrix24 automation adds a follow-up task or activity: “Verify work progress.”
•	Supervisors can monitor updates in real time through the CRM timeline.
🔹 Stage 5 — Completed
•	The agent clicks Job Completed in WhatsApp.
•	ChatBlend updates UF_CRM_FIELD_STATUS = Done.
•	Bitrix24 automatically moves the Deal to Completed.
•	Optionally, ChatBlend sends a closing message thanking the client or requesting feedback.
•	A CRM form (“Contact Information Form for use with Open Channels”) may be triggered to verify client data.
🔹 Stage 6 — Canceled (Exception Scenarios)
•	No Body on Site: Trigger: No check-in or location received within a defined time (e.g., 60 minutes after Scheduled).
Actions:
Move Deal to Canceled – No Body on Site.
Send an alert via WhatsApp.
Create a Bitrix24 follow-up task.

•	 Client Cannot Attempt: Trigger: Client cancels or replies “Unavailable.”
Actions:
Move Deal to Canceled – Client Cannot Attempt.
Update status and send a confirmation message.
Create a task for rescheduling.

🔹 Stage 7 — Reporting & Analytics
ChatBlend: Measures engagement — check-in times, completion rates, photo uploads, and WhatsApp response ratios.
Bitrix24: Tracks operational KPIs — completed vs. canceled visits, technician performance, SLA compliance, and location-based reporting.

5️⃣ Field Operations Workflow – Technical Visit
(Bitrix24 + ChatBlend Configuration)
1️⃣ In Bitrix24 — Workflow Automation Setup
Objective: configure triggers and rules so each event (check-in, location, completion, etc.) moves the Deal automatically.
Steps:
1.	Go to CRM → Deals → Pipeline → Automation Rules.
2.	Add rules for each stage:
o	Scheduled: ChatBlend Action → Send WhatsApp Template FieldOps_Visit.
o	On-Site: Field Changed → UF_CRM_FIELD_STATUS = Arrived → Move to On-Site.
o	In-Progress: Field Changed → update notes/photos → Create Activity “Verify Work Progress.”
o	Completed: Field Changed → UF_CRM_FIELD_STATUS = Done → Move to Completed and trigger closing message.
o	Canceled: Time-based trigger (no check-in after 60 minutes) → Move to Canceled.
3.	Add optional notifications (e.g., Slack/Email/CRM alerts) for supervisor visibility.

7️⃣ Webhook and API Extension (Advanced)

ChatBlend IA Bot / Flow

🎯 Objective
The ChatBlend IA Bot / Flow architecture enables companies to automate and centralize their customer interactions across WhatsApp Business API (WABA) and any connected CRM.
This design eliminates manual intervention in common service processes — such as plan renewals, ticket status checks, and follow-up notifications — by connecting ChatBlend’s intelligent chat flows with the CRM’s native REST API via secure webhooks.
Through this automation:
•	Clients receive instant, personalized responses on WhatsApp.
•	CRMs are automatically updated or queried in real time.
•	Agents are notified only when human interaction is required.
The result is a seamless, two-way communication loop between customer, ChatBlend IA Bot / Flow, and CRM, improving operational efficiency, customer satisfaction, and overall service continuity.
















🧭 Operational Guide — ChatBlend IA Bot / Flow
1️⃣ Entry Point: WhatsApp
1.	The client opens WhatsApp and sees two buttons: Renew Plan and Check Ticket.
2.	The client clicks one (or sends an equivalent message).
3.	The event from WhatsApp Business API (WABA) reaches the configured inbound webhook.________________________________________
2️⃣    Core Engine: ChatBlend IA Bot / Flow
4.	ChatBlend IA Bot / Flow receives the event and detects which option was selected.
5.	It splits the logic into two branches:________________________________________
3️⃣ Branch A — Renew Plan
6.	The ChatBlend outbound webhook triggers a call to the CRM webhook for plan renewal:
o	In ChatBlend (Developers → Webhooks), configure your CRM destination as:
https://your.company.com/webhook/crm-renewal
7.	The CRM webhook receives the call and runs its internal REST logic:
o	Updates the Deal/Task (e.g., STAGE_ID=NEW, adds comment, etc.).
8.	The CRM replies with a success response (e.g., { "result": true }).
9.	ChatBlend sends the user a confirmation via WhatsApp:
o	“✅ Your renewal has been successfully registered.”
10.	Flow ends in ChatBlend IA Bot / Flow.________________________________________
4️⃣ Branch B — Check Ticket
11.	The ChatBlend outbound webhook calls the CRM webhook for data retrieval:
o	In ChatBlend (Developers → Webhooks), configure:
https://your.company.com/webhook/crm-query
12.	The CRM webhook runs an internal REST GET request:
o	Fetches fields like TITLE, STAGE_ID, UF_CRM_DUE_DATE.
13.	The CRM returns the data in JSON format.
14.	ChatBlend processes and formats the response, then replies to the user:
o	“🔍 Your service status:
• Title: {{TITLE}}
• Stage: {{STAGE_ID}}
• Due date: {{UF_CRM_DUE_DATE}}”
15.	Flow ends in ChatBlend IA Bot / Flow.________________________________________
5️⃣ Webhook Pairing (Match)
•	CRM → Outbound Webhook: Create it first inside your CRM (e.g., Bitrix24 or another). It generates a URL.
•	ChatBlend → Outbound Webhook: Paste that CRM URL in ChatBlend (Developers → Webhooks).
o	Renewal: https://your.company.com/webhook/crm-renewal
o	Query: https://your.company.com/webhook/crm-query
Core idea: ChatBlend Webhook → CRM Webhook → CRM REST API.
The CRM performs the actual update or query internally.________________________________________
